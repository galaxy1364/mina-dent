\# mina-dent Folder Map (Lockpack)
<div dir="rtl">



این فایل برای اینه که اگر ویندوز/پروژه پاک شد و دوباره از صفر شروع کردیم، دقیق بدونیم هر پوشه برای چیه.



\## پوشه‌های سطح C:\\work



\- mina-dent\_WORK

  - معنی: محل کار روزانه (همیشه اینجا تغییر می‌دیم)

  - قانون: فقط این پوشه دستکاری می‌شود. بقیه فقط مرجع/بکاپ هستند.



\- mina-dent\_BASE

  - معنی: پایه تمیز (کلون/نصب پایه، بدون تغییرات کاری)

  - قانون: فقط برای شروع دوباره یا مقایسه. داخلش کدنویسی نکن.



\- mina-dent\_GOLDEN

  - معنی: نسخه طلایی (یک commit که CI کامل PASS شده)

  - قانون: فقط وقتی CI سبز شد و Evidence کامل داشتیم آپدیت می‌شود.



\- mina-dent\_LOCKED

  - معنی: نسخه قفل (کپی Read-only از GOLDEN یا PASS)

  - قانون: دستکاری ممنوع. فقط برای بازگشت سریع/مقایسه.



\- mina-dent (اگر وجود دارد)

  - معنی: نام عمومی/قدیمی (اگر استفاده نمی‌شود، فقط مرجع)

  - قانون: اگر WORK داریم، این نباید محل تغییر باشد.



\## داخل ریپو mina-dent\_WORK



\- LOCKPACK/

  - معنی: قوانین، چک‌لیست‌ها، evidence، و فایل‌های کنترل PASS/STOP

  - باید: FOLDER\_MAP.md + QG.json + (بعداً) BACKUP\_MANIFEST.json



\- QG.json

  - معنی: وضعیت گیت‌ها (PASS/FAIL/PENDING) + لینک/اسم evidence

  - قانون: بدون QG.json کامل، هیچ PROCEED نداریم.



\- node\_modules/

  - معنی: نصب محلی

  - قانون: داخل بکاپ ZIP نباید باشد (خیلی بزرگ + غیرقابل اعتماد). همیشه با npm ci قابل ساخت است.



\## قانون بکاپ

\- بکاپ ZIP باید همیشه داخل خودش این فایل‌ها را داشته باشد:

  - LOCKPACK/FOLDER\_MAP.md

  - LOCKPACK/BACKUP\_MANIFEST.json (وقتی ساختیم)

  - QG.json

\- نام پیشنهادی بکاپ:

  mina-dent\_WORK\_PASS\_YYYYMMDD\_HHMMSS.zip
